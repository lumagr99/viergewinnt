Detailed Description

Die ConnectFour-Klasse enthält die Spiellogik für ein Vier Gewinnt Spiel. Die Klasse enthält die Tischplatte, das Spielbrett und die 42 Spieltoken.

Die draw-Methode zeichnet die enthaltenen Objekte und benötigt einen aktiven OpenGL-Kontext. Über die Methode selectToken kann ein Token ausgewählt und anschließend mit der Methode moveToken bewegt werden. Mithilfe der Methode deselectToken kann das ausgewählte Token deselektiert werden.

Das Spiel startet mit dem im Konstruktor übergebenen Spieler. Bei Spielende wird ein gameOver Signal mit Gewinnerfarbe emittiert.

Bei folgenden Aktionen wird ein soundRequested Signal mit dem Dateinamen der abzuspielenden Datei emittiert:

  • Bei Selektion eines Tokens
  • Beim Einfügen eines Tokens in das Spielbrett
  • Beim Spielerwechsel
  • Beim Spielende

void ConnectFour::descentAnimation ( )

Animiert den Abstieg eines eingefügten Tokens zur Zielposition.

◆ jumpDownAnimation()

void ConnectFour::jumpDownAnimation ( )

Animiert den Sprung des selektierten Tokens von einer Einfügeposition zur Tischplatte.

Das Token wird durch die Animation um 90° um die X-Achse gedreht und das Flag m_rotated auf false gesetzt.

Die Methode wird alle 16 msec durch die Methode doSynchronizeThreads der Szene aufgerufen, falls das Flag m_animateJumpDown true ist.

Ist m_animationStep = ANIMATION_STEPS wird das Flag m_animateJumpDown auf false gesetzt.

Die Anzahl der Animationsschritte entspricht ANIMATION_STEPS.

◆ moveToken()

void ConnectFour::moveToken ( const QVector3D &  vMove)

Bewegt das selektierte Token entsprechend der Vier Gewinnt Spielregeln.

Bei Kollision mit anderen Spielsteinen wird die Bewegung verhindert.

Bei Kollision mit dem Spielbrett wird das Flag m_animateJumpUp auf true gesetzt, was dazu führt dass eine Animation zu einer möglichen Einfügeposition ausgeführt wird.

Wird das Token von einer Einfügeposition wegbewegt, so wird das Flag m_animateJumpDown auf true gesetzt und dadurch eine Animation zur Tischplatte ausgeführt.

Parameters
vMoveVektor in dessen Richtung das Token bewegt werden soll.
